## 42. 接雨水
- **标签：** 双指针 / 数组
- **原题链接：** [点击跳转](https://leetcode.cn/problems/trapping-rain-water/?envType=study-plan-v2&envId=top-100-liked)

### 💡 思路历程
依旧忘得一干二净，只知道可以用单调栈做，但怎么做就不知道了，个人认为开两个数组记录左右两边最大高度的思路（dp）更容易理解与记忆，单调栈其实也很简洁，双指针法相对难理解，其实是一个动态调整的过程，因为只需要看左右两边相对短的那根最长柱就好了，无需同时知道
两个，所以无需知道全局。代码都放下面。

### 💻 C++ dp法代码实现
``` cpp
class Solution {
public:
    int trap(vector<int>& height) {
        vector<int> LeftMax(height.size(),0);
        vector<int> RightMax(height.size(),0);
        int sum = 0;
        LeftMax[0] = height[0];
        for(int i = 1;i<height.size();i++){
            LeftMax[i] = max(height[i-1],LeftMax[i-1]);
        }
        LeftMax[height.size()-1] = height[height.size()-1];
        for(int i = height.size()-2;i>=0;i--){
            RightMax[i] = max(height[i+1],RightMax[i+1]);
        }
        for(int i = 1;i<height.size()-1;i++){
            int h = min(RightMax[i],LeftMax[i]) - height[i];
            if(h>0) sum+=h;
        }
        return sum;
    }
};
```
### 💻 C++ 双指针法代码实现
``` cpp
class Solution {
public:
    int trap(vector<int>& height) {
        int n = height.size();
        int left = 0;
        int right = n-1;
        int leftmax = 0,rightmax = 0;
        int result = 0;
        while(left<right){
            leftmax = max(leftmax,height[left]);
            rightmax = max(rightmax,height[right]);
            if(leftmax<rightmax) {
                result += leftmax-height[left];
                left++;
            }
            else {
                result += rightmax - height[right];
                right--;
            }
        }
        return result;
    }
};
```
### 💻 C++ 单调栈法代码实现
``` cpp
class Solution {
public:
    int trap(vector<int>& height) {
        stack<int> st;
        st.push(0);
        int sum = 0;
        for(int i = 1;i<height.size();i++){
            while(!st.empty()&&height[i]>=height[st.top()]){
                int mid = st.top();
                st.pop();
                if(!st.empty()){
                    int h = min(height[st.top()],height[i])-height[mid];
                    int width = i-st.top()-1;
                    sum +=h*width;
                }
                
            }
            st.push(i);
        }
        return sum;
    }
};
```
### 值得注意
三种方法全部掌握，也算是通关接雨水了。前两种方法是纵向算雨水，单调栈则是横向算雨水，但其实并非标准矩形那样平行上去，用二重积分的思想就很好理解了。其中单调栈做法中的while循环里，把>改为>=，会加快运行速度，为什么呢？就算提前弹出来不是也会进入if判断里面再算h跟w吗？**原因如下**：

1.这样可以保持栈的深度一直较浅，前几个元素极大概率会一直在cpu的L1缓存中，速度极快，深栈自然很可能cache装不完，就得去访问内存，自然速度较慢。本质上提升了缓存命中率（死去的计组）；

2.还有就是分支预测的优化，当你使用 >= 时，对于等高的柱子，你的 while 循环几乎总是“运行一次就结束”。这种高度可预测的模式让 CPU 的**指令流水线（Pipeline）**跑得极其顺滑，减少了分支预测失败带来的性能损耗。反之，如果在数组末尾突然出现一个巨大的 while 循环，CPU 可能会在这里产生多次预测失败（Pipeline Stall），从而拖慢执行时间。
