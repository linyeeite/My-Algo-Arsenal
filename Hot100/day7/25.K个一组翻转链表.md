## 25.K个一组翻转链表
- **标签：** 链表
- **原题链接：** [点击跳转](https://leetcode.cn/problems/reverse-nodes-in-k-group/description/?envType=study-plan-v2&envId=top-100-liked)

### 💡 思路历程
手撕成功，用的是迭代法，递归确实没能想明白，但其实跟两两交换那题非常相似，也有点像先序遍历和中序遍历那样，先处理本段逻辑，再处理下面逻辑，还是自上向下的，链表的话在于原本head结点会成为本段最后一个结点，然后要接后面处理的链表，很适合递归逻辑。
```cpp
class Solution {
private: 
    bool is_vaild(ListNode* head,int k){
        ListNode* cur = head->next;
        while(k--){
            if(cur==nullptr) return false;
            cur = cur->next;
        }
        return true;
    }
    ListNode* reverse(ListNode* head,int k){
        ListNode *cur = head->next,*Next;
        ListNode* last = cur;
        head->next = nullptr;
        while(k--){
            Next = cur->next;
            cur->next = head->next;
            head->next = cur;
            cur = Next;
        }
        last->next = Next;
        return last;
    }
public:
    ListNode* reverseKGroup(ListNode* head, int k) {
        ListNode* dummy = new ListNode();
        dummy->next = head;
        ListNode* cur = dummy;
        while(is_vaild(cur,k)){
            cur = reverse(cur,k);
        }
        head = dummy->next;
        delete(dummy);
        return head;
    }
};
```
### 🔥值得注意
在这里自述一下本体逻辑。使用单调队列，队头为对应窗口的最大值，要适应滑动窗口的移动，我们要自定义一个队列，即用deque这个底层容器，队列其实只要放可以成为最大元素的值就好了。

push时，如果本元素大于前面的元素，自然弹出前面的，**直到碰到不比自己小的**，然后再把自己push_back,即放到队尾。

pop时，因为我们在push时是有选择的push的，有些已经被弹出了，所以一个很自然的逻辑就是，只有队头元素是自己时，才弹出，说明窗口往前移动，本元素已经不在窗口里了，而本元素又是该窗口未移动前的最大元素，故弹出。这些逻辑用if判断就好了，使得后续逻辑好写。

再来个取队头元素的函数即可。

然后初始化先将前k个元素放入队列，将队头元素放入结果里，然后才从第k+1个开始，pop第前k个元素，push当前元素，再取队头元素加入结果集。遍历完即可。
