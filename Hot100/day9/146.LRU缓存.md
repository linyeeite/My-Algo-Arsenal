## 146.LRU缓存
- **标签：** 双向链表+哈希表
- **原题链接：** [点击跳转](https://leetcode.cn/problems/lru-cache/description/?envType=study-plan-v2&envId=top-100-liked)

### 💡 思路历程
几乎无思路。重述下标准思路：用一个双向链表和哈希表完成，双向链表用于按使用时间排序结点，哈希表用于记录key值对应的结点指针，以达到O（1）的时间复杂度，巧妙之处在于用上头尾哨兵结点，统一处理。然后是时间上的优化，首先注意如果要多次用到哈希表key对应的val值，

一定要提前取出来，这样快很多，否则要查表多次。
### 💻 C++ 递归极简代码实现
```cpp
 struct Node{
    int key,value;
    Node *prev,*next;
    Node(int k = 0,int v = 0): key(k),value(v),prev(nullptr),next(nullptr){}
};


class LRUCache {
private:
    int capacity;
    unordered_map<int,Node*> cache;
    Node *head,*tail;


    void moveToHead(Node* node){
        remove(node);
        addToHead(node);
    }

    void remove(Node* node){
        node->prev->next = node->next;
        node->next->prev = node->prev;
    }

    void addToHead(Node* node){
        node->next = head->next;
        node->prev = head;
        node->next->prev = node;
        head->next = node;
    }
public:
    LRUCache(int capacity):capacity(capacity){
        head = new Node();
        tail = new Node();
        head->next = tail;
        tail->prev = head;
    }
    
    int get(int key) {
        if(cache.find(key)==cache.end()) return -1;
        auto node = cache[key];
        moveToHead(node);
        return node->value;
    }
    
    void put(int key, int value) {
        if(cache.find(key) != cache.end()){
            auto node = cache[key];
            node->value = value;
            moveToHead(node);
        }
        else{
            if(cache.size() == capacity){
                Node* discarded = tail->prev;
                cache.erase(discarded->key);
                remove(discarded);
                delete(discarded);
                
            }
            Node* newnode = new Node(key,value);
            cache[key] = newnode;
            addToHead(newnode);
            
        }
        
    }
};

/**
 * Your LRUCache object will be instantiated and called as such:
 * LRUCache* obj = new LRUCache(capacity);
 * int param_1 = obj->get(key);
 * obj->put(key,value);
 */
```
### 🔥值得注意
LRUCache(int capacity):capacity(capacity)  

其实理解成构造类的函数就好了。

这种写法在 C++ 中被称为 构造函数初始化列表 (Constructor Initialization List)

冒号表示初始化列表的开始。第一个 capacity：指的是类中的成员变量（Member Variable）。括号里的 capacity：指的是构造函数的入参（Parameter）。

成员变量初始化的顺序，只取决于它们在类里声明的先后顺序，跟你写在冒号后面的顺序无关。所以大家一般习惯按照声明顺序来写初始化列表，这样逻辑最清晰。
