## 76.最小覆盖字串
- **标签：** 滑动窗口/数组
- **原题链接：** [点击跳转](https://leetcode.cn/problems/minimum-window-substring/submissions/709244226/?envType=study-plan-v2&envId=top-100-liked)

### 💡 思路历程
原本想记录t中字符在s中的对应位置，不仅要遍历很多遍，而且后续找区间也很麻烦。用两个map记录对应需求量和已有量，然后滑动窗口，右边为了达到要求而化，左边是一旦达到要求，就收缩，再还能达到要求的前提下持续收缩，达不到了就让右窗口继续扩大，一旦达到就记录
对应长度，若小于此前遍历的长度，就更新对应位置，但无论是否小于与否，都得收缩。
### 💻 C++ 代码实现
```cpp
class Solution {
public:
    string minWindow(string s, string t) {
        unordered_map<char,int> need,window;
        int vaild = 0;
        for(char t1:t){
            need[t1]++;
        }
        int start = 0,len = INT_MAX;
        int left = 0,right = 0;
        while(right<s.size()){
            char s1 = s[right];
            right++;
            if(need.count(s1)){
                window[s1]++;
                if(need[s1]==window[s1]) vaild++;
            }
            while(vaild ==need.size()){
                if(right-left<len){
                    start = left;
                    len = right-left;
                }
                char c = s[left];
                left++;
                
                if(need.count(c)){
                    if(need[c]==window[c]){
                        vaild--;
                    }
                    window[c]--;
                }
            }
            
        }
        return len==INT_MAX?"":s.substr(start,len);
    }
};
```
### 🔥值得注意
逻辑要很清晰，用了一个vaild记录是否达标，有点像找到字母异位词那的diff，然后right的移动逻辑和left的移动逻辑，start和len的更新逻辑也要很清晰。估计很快忘。。
